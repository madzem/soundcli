Controls your browser

soundcli never streams audio. Your logged-in SoundCloud tab plays the music — full tracks, your account, ads as usual — and soundcli drives it over MPRIS / D-Bus.

Queue browser

Pass a set URL and soundcli fetches the tracklist. Scroll the whole queue, select any track, and jump straight to it with Enter.

Keyboard-driven

No mouse needed. Every action has a key — play/pause, next/prev, seek, volume, and quit — rendered in a clean Ratatui interface.

Native Rust binary

A single small binary, no runtime to install. Ships prebuilt as .deb, .rpm, a portable tarball, and Homebrew. Linux (needs D-Bus; PipeWire for volume).

INSTALL — NO RUST OR COMPILER NEEDED

Debian · Ubuntu · Mint

.deb package
$ sudo apt install ./soundcli_*_amd64.deb

Fedora · RHEL · openSUSE

.rpm package
$ sudo dnf install ./soundcli-*.x86_64.rpm

Any distro

portable binary
$ tar -xzf soundcli-*-linux-gnu.tar.gz
$ sudo install -m755 soundcli /usr/local/bin/

Homebrew (Linux)

Linuxbrew
$ brew install madzem/soundcli/soundcli

Cargo

if you already have Rust
$ cargo install --git \
    https://github.com/madzem/soundcli

Grab packages from the Releases page. Full step-by-step instructions — including checksum verification — are in INSTALL.md. soundcli is Linux-only: it needs a D-Bus session (any standard desktop) and, for in-app volume, PipeWire's wpctl.

USAGE
$ soundcli --playlist soundcloud.com/<owner>/sets/<name> # open a set & control it
$ soundcli # control whatever SoundCloud tab is already playing
$ soundcli --demo # try the UI with sample data, no browser
$ soundcli --list-players # list MPRIS players (diagnose detection)
$ soundcli --dump-queue <set-url> # print the fetched tracklist
$ soundcli --help
